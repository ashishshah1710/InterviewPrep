Java OOP & Constructors

Object-oriented fundamentals and constructor rules commonly tested at 2–3 YOE level.

What is a Constructor?

OOP

A constructor is a special block of code invoked when an object is created with new. It initializes instance variables and prepares the object for use. Same name as the class, no return type (not even void).

public class Employee {
    private final String name;
    private double salary;

    // Default constructor (provided by compiler if none written)
    public Employee() {
        this.name = "Unknown";
    }

    // Parameterized constructor
    public Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
    }
}

If you define any constructor, Java does not provide a default no-arg constructor unless you write one explicitly.

Can we have multiple constructors in a class?

OOP

Yes — this is called constructor overloading. Multiple constructors in the same class must differ by parameter list (number, type, or order).

public class Order {
    public Order() { this("GUEST"); }
    public Order(String customerId) { this(customerId, List.of()); }
    public Order(String customerId, List<LineItem> items) {
        // initialize
    }
}

Use this(...) to chain constructors and avoid duplicate initialization logic. Only one this() or super() call allowed as the first statement in a constructor.

Why can't constructors be overridden?

OOP

Constructors are not inherited and therefore cannot be overridden — overriding requires inheritance of an instance method from a parent class.

  • Constructors belong to the class that defines them; subclass gets its own constructors.
  • Subclass constructor must call a parent constructor via super(...) before its own logic.
  • What looks like "overriding" is actually overloading (same class, different params) or defining a new constructor in a subclass.
  • Constructors are not polymorphic — the compiler binds new SubClass() to SubClass's constructor at compile time.
