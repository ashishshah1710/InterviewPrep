Two Sum Problem

DSA HashMap

Given an array of integers nums and a target target, return indices of the two numbers that add up to target. Assume exactly one solution exists.

Approach: Use a HashMap to store each number and its index as you iterate. For each element, check if target - current already exists in the map. Time O(n), space O(n).

public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[] { map.get(complement), i };
        }
        map.put(nums[i], i);
    }
    throw new IllegalArgumentException("No solution");
}

// Example: nums = [2, 7, 11, 15], target = 9 → [0, 1]
Follow-ups: Sorted array → two-pointer O(n). All pairs summing to target → store frequencies. Return values instead of indices → return pair from map lookup.

Sort a list of employees: salary descending, experience ascending

Java Streams Comparator 
record Employee(String name, double salary, int experienceYears) {}

List<Employee> sorted = employees.stream()
    .sorted(Comparator
        .comparing(Employee::salary).reversed()
        .thenComparing(Employee::experienceYears))
    .toList();

// Without streams
employees.sort(Comparator
    .comparingDouble(Employee::salary).reversed()
    .thenComparingInt(Employee::experienceYears));

Key point: reversed() applies only to salary. Experience stays ascending via thenComparing. For stable sort when all fields tie, add .thenComparing(Employee::name).
