Design a ConcurrentHashMap from scratch

LLD Concurrency

Core design: array of buckets, each bucket is a linked list. Writes synchronize on the bucket head; reads are mostly lock-free.

public class SimpleConcurrentHashMap<K, V> {
    private static final int DEFAULT_CAPACITY = 16;
    private final Node<K, V>[] buckets;
    private final ReentrantLock[] locks;

    static class Node<K, V> {
        final K key;
        volatile V value;
        volatile Node<K, V> next;

        Node(K key, V value) {
            this.key = key;
            this.value = value;
        }
    }

    @SuppressWarnings("unchecked")
    public SimpleConcurrentHashMap(int capacity) {
        this.buckets = new Node[capacity];
        this.locks = new ReentrantLock[capacity];
        for (int i = 0; i < capacity; i++) {
            locks[i] = new ReentrantLock();
        }
    }

    private int index(K key) {
        return Math.abs(key.hashCode()) & (buckets.length - 1);
    }

    public V get(K key) {
        int idx = index(key);
        Node<K, V> node = buckets[idx]; // volatile read
        while (node != null) {
            if (node.key.equals(key)) return node.value;
            node = node.next;
        }
        return null;
    }

    public void put(K key, V value) {
        int idx = index(key);
        locks[idx].lock();
        try {
            Node<K, V> node = buckets[idx];
            while (node != null) {
                if (node.key.equals(key)) {
                    node.value = value; // update existing
                    return;
                }
                node = node.next;
            }
            // insert at head
            Node<K, V> newNode = new Node<>(key, value);
            newNode.next = buckets[idx];
            buckets[idx] = newNode;
        } finally {
            locks[idx].unlock();
        }
    }
}

Key design decisions: Per-bucket locking (not global lock), volatile fields for visibility, no null keys/values, resize requires locking all buckets or using a separate resize lock.

Design a Parking Lot System

LLD State Pattern 
enum VehicleType { MOTORCYCLE, CAR, BUS }
enum SpotType    { COMPACT, LARGE, HANDICAPPED }

class Vehicle {
    private final String licensePlate;
    private final VehicleType type;
}

class ParkingSpot {
    private final String id;
    private final SpotType type;
    private final int floor;
    private volatile boolean occupied;
}

class Ticket {
    private final String ticketId;
    private final String spotId;
    private final LocalDateTime entryTime;
    private final Vehicle vehicle;
}

interface FeeStrategy {
    double calculateFee(LocalDateTime entry, LocalDateTime exit);
}

class ParkingLot {
    private final Map<SpotType, Queue<ParkingSpot>> availableSpots;
    private final FeeStrategy feeStrategy;

    synchronized Ticket park(Vehicle vehicle) {
        SpotType needed = mapVehicleToSpot(vehicle.getType());
        ParkingSpot spot = availableSpots.get(needed).poll();
        if (spot == null) throw new ParkingFullException();
        spot.setOccupied(true);
        return new Ticket(generateId(), spot.getId(),
                          LocalDateTime.now(), vehicle);
    }

    synchronized double unpark(Ticket ticket) {
        double fee = feeStrategy.calculateFee(
            ticket.getEntryTime(), LocalDateTime.now());
        // release spot back to queue
        return fee;
    }
}

Design points: Motorcycle can use compact/large/handicapped; car uses compact/large; bus uses large only. Singleton for ParkingLot. Synchronized spot allocation. Strategy pattern for hourly vs flat fee.

Design a Rate Limiter

LLD Design Patterns 
// Core interface
interface RateLimiter {
    boolean allowRequest(String clientId);
    int remainingTokens(String clientId);
}

// Token Bucket implementation
class TokenBucketRateLimiter implements RateLimiter {
    private final int capacity;
    private final double refillRatePerSecond;
    private final Map<String, Bucket> buckets = new ConcurrentHashMap<>();

    private static class Bucket {
        double tokens;
        long lastRefillTimestamp;

        synchronized boolean tryConsume(int n, int capacity,
                                        double refillRate) {
            refill(capacity, refillRate);
            if (tokens >= n) {
                tokens -= n;
                return true;
            }
            return false;
        }

        private void refill(int capacity, double refillRate) {
            long now = System.nanoTime();
            double elapsed = (now - lastRefillTimestamp) / 1_000_000_000.0;
            tokens = Math.min(capacity, tokens + elapsed * refillRate);
            lastRefillTimestamp = now;
        }
    }

    public boolean allowRequest(String clientId) {
        Bucket bucket = buckets.computeIfAbsent(
            clientId, k -> new Bucket(capacity));
        return bucket.tryConsume(1, capacity, refillRatePerSecond);
    }
}

// Factory for different tiers
class RateLimiterFactory {
    static RateLimiter create(RateLimitTier tier) {
        return switch (tier) {
            case FREE    -> new TokenBucketRateLimiter(10, 1.0);
            case PREMIUM -> new TokenBucketRateLimiter(100, 10.0);
            case ENTERPRISE -> new SlidingWindowRateLimiter(1000, 60);
        };
    }
}

Algorithms: Token Bucket (smooth rate, allows bursts), Leaking Bucket (constant output rate), Fixed Window (simple but boundary burst problem), Sliding Window Log (precise, memory-heavy), Sliding Window Counter (Redis-based, distributed).

Patterns: Strategy (swap algorithm), Factory (create by tier), Decorator (add logging/metrics), Singleton (shared limiter instance).

Ensure thread safety and immutability in your design

Immutability Thread Safety 
// Immutable value object
public final class Money {
    private final BigDecimal amount;
    private final String currency;

    public Money(BigDecimal amount, String currency) {
        this.amount = amount;
        this.currency = Objects.requireNonNull(currency);
    }

    public Money add(Money other) {
        assertSameCurrency(other);
        return new Money(this.amount.add(other.amount), this.currency);
    }

    // No setters — new instance for every change
}

// Defensive copying for mutable fields
public final class Order {
    private final String id;
    private final List<LineItem> items;
    private final LocalDateTime createdAt;

    public Order(String id, List<LineItem> items) {
        this.id = id;
        this.items = List.copyOf(items); // immutable copy
        this.createdAt = LocalDateTime.now();
    }

    public List<LineItem> getItems() {
        return items; // already immutable
    }
}
  • Declare class as final to prevent subclassing that breaks immutability
  • All fields private final
  • No setters — return new instances for changes
  • Defensive copy mutable inputs in constructor; return unmodifiable views from getters
  • Use List.of(), Map.of(), or Guava immutable collections

Deep vs. Shallow Cloning: When does it matter?

Cloning

Primitive fieldsCopied by valueCopied by value

Default Object.clone()ShallowMust override
MemoryLessMore
Safety with mutable nested objectsUnsafe — changes affect bothSafe — fully independent
class Address { String city; }
class Person implements Cloneable {
    String name;
    Address address; // mutable reference type

    @Override
    protected Person clone() throws CloneNotSupportedException {
        Person copy = (Person) super.clone(); // shallow
        copy.address = new Address();          // deep copy nested
        copy.address.city = this.address.city;
        return copy;
    }
}

When it matters: When nested objects are mutable (Lists, Maps, custom objects). Shallow clone is fine when all fields are primitives or immutable (String, Integer, LocalDate). Prefer copy constructors or serialization libraries (Jackson, Apache Commons) for reliable deep copies.

Design a Vending Machine

LLD State Pattern 
// State interface
interface VendingMachineState {
    void insertCoin(VendingMachine vm, int amount);
    void selectProduct(VendingMachine vm, String productId);
    void dispense(VendingMachine vm);
    void cancel(VendingMachine vm);
}

class IdleState implements VendingMachineState {
    public void insertCoin(VendingMachine vm, int amount) {
        vm.addBalance(amount);
        vm.setState(new HasMoneyState());
    }
    // selectProduct and dispense throw IllegalStateException
}

class HasMoneyState implements VendingMachineState {
    public void selectProduct(VendingMachine vm, String productId) {
        Product product = vm.getInventory().get(productId);
        if (product == null) throw new ProductNotFoundException();
        if (vm.getBalance() < product.getPrice())
            throw new InsufficientFundsException();
        vm.setSelectedProduct(product);
        vm.setState(new DispensingState());
        vm.dispense();
    }
}

class VendingMachine {
    private VendingMachineState state = new IdleState();
    private final Map<String, Product> inventory;
    private int balance;
    private Product selectedProduct;

    void insertCoin(int amount) { state.insertCoin(this, amount); }
    void selectProduct(String id) { state.selectProduct(this, id); }
    void dispense() { state.dispense(this); }
    void setState(VendingMachineState state) { this.state = state; }
}

State transitions: Idle → HasMoney (coin inserted) → Dispensing (product selected) → Idle (product dispensed, change returned).

Patterns used: State (behavior changes by machine state), Strategy (payment method — coin, card, UPI), Singleton (one machine instance), Observer (low-inventory alert to admin).

Thread safety: Synchronize state transitions and inventory decrements to prevent double-dispense in concurrent access.
